# VK-бот «Жадный Мир»

Асинхронный бот сообщества VK для реестра карт, анкет персонажей, Шакеев и
вспомогательного кубика. Интерфейсы физически разделены: пошаговые сценарии ЛС
находятся в `bot/handlers/dm/`, команды беседы - в `bot/handlers/chat/`.

## Возможности MVP

- ЛС: выбор одной из своих анкет, постраничные реестры карт и анкет, поиск
  карты, перевод Шакеев от имени выбранного персонажа.
- Админ-панель в ЛС сначала предлагает выбрать раздел «Анкеты» или «Карты».
  Внутри раздела можно добавить новую запись либо выбрать существующую; действия
  редактирования, удаления и операции с Шакеями появляются после выбора записи.
  Для начисления или списания Шакеев достаточно указать только сумму.
- AI-импорт в админ-панели: исходную анкету можно прислать несколькими
  сообщениями и дополнить изображениями внешности или книги. Модель переносит
  только предоставленные сведения без дополнений, результат показывается для
  проверки и записывается в базу лишь после подтверждения. При создании или
  редактировании Контура AI получает заранее выбранные реальные карты и не может
  менять его состав.
- Отдельный режим `AI-Ассистент` принимает свободные просьбы администратора,
  сам ищет анкеты, карты, Контуры и историю Шакеев, а любые изменения сначала
  оформляет как постоянный журналируемый план. План выполняется атомарно только
  после подтверждения; удаления требуют второго подтверждения.
- Один VK ID может владеть несколькими анкетами; новые анкеты добавляет только
  администратор.
- Администратор открывает персонажа из реестра либо находит его по имени,
  `#ID` анкеты или VK ID владельца. Если одному VK ID принадлежат несколько
  персонажей, бот показывает отдельный выбор и не подставляет первого попавшегося.
- В карточке карты и анкете администратор получает быстрые кнопки редактирования
  и удаления. Удаление выполняется только после отдельного подтверждения;
  для обычных пользователей реестры и собственные анкеты доступны только для чтения.
- Кнопка `Создать бэкап БД` в админ-панели делает согласованный SQLite-снимок,
  проверяет его целостность и отправляет администратору `.db`-файл документом VK.
- Добавление карты начинается с выбора типа: Особая, Заклинание, Обычная,
  Контурная или ГеймМастерская. После этого администратор выбирает пошаговое
  заполнение, готовый многострочный шаблон или оформление свободного описания
  через AI. В пошаговом режиме редкость, подтип Контурной карты и лимит Особой
  карты выбираются кнопками.
- Особые карты используют отдельные игровые номера 0–99. Заклинания и
  Контурные карты используют общий пул номеров от 0. Обычные карты не занимают
  номер и не создаются в реестре — их физические копии добавляются сразу
  выбранному персонажу. Карты ГеймМастеров остаются служебными и не выдаются.
- Карты персонажа в ЛС разделены на Особые, Заклинания, Контурные и Обычные.
  Владелец анкеты и администратор могут выгрузить их в `.xlsx` с четырьмя
  листами и датами получения. Администратор может выгрузить реестр в `.xlsx`
  с отдельными листами Особых, Заклинаний и Контурных карт.
- Беседа: `?карта`, `?профиль`, `?визитка`, `?кубик`, `?использовать`, `?помощь`.
- Реальные копии карт хранятся в `CardOwnership`; у персонажа может быть несколько
  копий одной карты. Связанную с Контуром копию нельзя забрать до изменения или
  разбора Контура.
- Заклинания и Обычные карты можно хранить стопками. Выдача, изъятие и расход
  принимают количество и затрагивают только указанное число свободных физических
  копий; игровые расходы сохраняются в отдельном журнале.
- Контуры доступны только владельцу анкеты и администраторам в ЛС. Анкета начинает
  с двух Контурных слотов; их количество растёт отдельно от размера каждого
  Контура. Размер — от 2 до 5 карт, в составе обязательна Контурная карта.
- Каждое движение Шакеев записывается в журнал и может быть сверено с балансом.
- У анкеты может быть несколько артов. Изображения сохраняются локально, один арт
  назначается основным и показывается вместе с `?профиль`. Владелец просматривает
  свою галерею в ЛС, а администратор добавляет изображения, меняет подписи,
  выбирает основной арт и удаляет изображения с подтверждением.

## Установка

Нужен Python 3.11 или новее. Excel-экспорт работает через Python-библиотеку
`openpyxl`, которая устанавливается вместе с остальными зависимостями.

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install -r requirements.txt
Copy-Item .env.example .env
```

Заполните в `.env` токен сообщества, ID группы и VK ID администраторов. Для
AI-импорта анкет, оформления карт и AI-редактирования Контуров также добавьте
ключ DS Lab. Он также используется режимом AI-Ассистента:

```dotenv
DSLAB_API_KEY=ваш-ключ-dslab
DSLAB_MODEL=deepseek-v4-flash
DSLAB_VISION_MODEL=gemini-2.5-flash-lite
DSLAB_MAX_TOKENS=4000
DSLAB_AGENT_MAX_TOKENS=8000
DSLAB_AGENT_TIMEOUT_SECONDS=180
```

Локальное хранилище артов настраивается отдельно:

```dotenv
CHARACTER_ART_STORAGE_DIR=storage/character_art
CHARACTER_ART_MAX_FILE_BYTES=20971520
CHARACTER_ART_MAX_TOTAL_BYTES=4294967296
CHARACTER_ART_MAX_PER_CHARACTER=50
```

По умолчанию разрешены JPEG, PNG и WebP до 20 МБ, до 50 изображений на анкету
и до 4 ГБ суммарно. Каталог `CHARACTER_ART_STORAGE_DIR` должен находиться на
постоянном диске хостинга. SQLite хранит метаданные и относительные пути, но не
содержимое изображений. Поэтому стандартный бэкап `.db` необходимо хранить
вместе с копией каталога артов.

Полный XLSX-экспорт анкеты содержит отдельный лист «Арты» с метаданными и
уменьшенными превью. Экспорт одних карт изображения не включает.

Графические визитки создаются самим ботом через Pillow и сохраняются отдельно:

```dotenv
PROFILE_CARD_STORAGE_DIR=storage/profile_cards
PROFILE_CARD_FONT_REGULAR=
PROFILE_CARD_FONT_BOLD=
```

Команды `?визитка` и `?карточка` показывают PNG с основным артом, именем,
рейтингом, Шакеями, статами, навыками, картами и Контурами. Бот хеширует данные
анкеты, её карты, Контуры и основной арт. Пока хеш не изменился, повторно
используются тот же локальный PNG и сохранённое VK-вложение. После изменения
исходных данных старая версия заменяется автоматически. На Windows используются
Segoe UI, на типичном Linux-хосте — DejaVu Sans; нестандартные пути можно задать
переменными `PROFILE_CARD_FONT_REGULAR` и `PROFILE_CARD_FONT_BOLD`.

`DSLAB_AGENT_MAX_TOKENS` ограничивает только один ответ AI-Ассистента, а не
его контекст или историю. Лимит вынесен в конфигурацию, чтобы длинные планы не
обрезались, но ошибочный запрос не мог бесконтрольно генерировать ответ.

Логи одновременно выводятся в терминал и сохраняются в `LOG_FILE`. Файл
ротируется по `LOG_MAX_BYTES`, старые копии ограничены `LOG_BACKUP_COUNT`.
Каждая задача AI-Ассистента получает `request_id`: по нему в логе видны все
обращения к DS Lab, длительность, расход токенов, проверка JSON, попытка
исправления, read-инструменты и создание плана. При `LOG_LEVEL=DEBUG` также
пишется обрезанный сырой ответ модели; ключи и токены конфигурации не логируются.

Присланный в этих сценариях текст передаётся внешнему сервису DS Lab и
текстовой модели DeepSeek. Если к анкете приложено изображение, бот
автоматически использует мультимодальную Gemini Flash Lite. Затем
создайте схему и запустите бота:

```powershell
python -m alembic upgrade head
python run.py
```

Для PostgreSQL достаточно заменить `DATABASE_URL`, например на
`postgresql://user:password@localhost/zhadny_mir`, и снова выполнить миграции.

## Команды беседы

```text
?карта Ясень
?профиль
?профиль Ава
?профиль 12
?визитка
?визитка Пикколо
?карточка 12
?кубик
?кубик 6
?кубик 1 20
?использовать [#ID анкеты] Название карты [xКоличество] @соигрок
?помощь
```

Кубик - последний довод для тупиковой сцены, а не замена тексту, позиции и
подготовке персонажей.

## Админские команды в ЛС

Основные операции запускаются постоянной клавиатурой VK под полем ввода.
Дополнительно доступны:

```text
?выдать <ID анкеты> <ID карты в БД> [количество]
?забрать <ID анкеты> <ID карты в БД> [количество]
?шакеи <ID анкеты> +100
?шакеи <ID анкеты> -100
```

Права для всего админского набора проверяет общее правило на лейблере по
`ADMIN_VK_IDS`, а не каждый обработчик отдельно. Исполнитель AI-Ассистента
дополнительно повторяет эту проверку перед каждым изменяющим инструментом.
